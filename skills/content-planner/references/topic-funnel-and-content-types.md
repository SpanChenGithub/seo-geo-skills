# Topic, Funnel, and Content Types

Use this reference when assigning the analyst-created `Topic`, `Funnel`, and `Content Type` fields. Treat these fields as reasoned classifications, not Ahrefs facts. Keep the classification consistent across the country workbook and explain edge cases in `Methodology`.

## Assign topics and site structure

- Name each `Topic` as a concise, stable cluster label in the target language.
- Group pages by the audience problem, job to be done, or product capability they jointly cover; do not group by a shared word alone.
- Identify a Pillar page for every important cluster and connect narrower Cluster pages beneath it.
- Keep the hierarchy clear enough for users and crawlers. Prefer a deeper coherent hierarchy over a shallow collection of unrelated pages.
- Link lower-level pages upward to their Pillar. Link laterally only where the next page genuinely advances the user's task or decision.
- Give strategically important pages more relevant internal links.
- Record page level, parent page, and link direction in `Topic Map` and `Roadmap`.

## Assign funnel stage

Assign exactly one primary funnel stage according to the dominant search intent:

| Funnel | User state | Common signals | Planning implication |
|---|---|---|---|
| `TOFU` | Becoming aware of a problem or topic | definitions, explanations, statistics, trends, broad examples | Educate, earn discovery, and introduce the next relevant cluster page. |
| `MOFU` | Exploring approaches and evaluating solution categories | workflows, use cases, templates, methods, category comparisons | Help the user understand options and move toward a suitable solution. |
| `BOFU` | Selecting, trying, or buying a product or a direct solution | tool/generator terms, best products, alternatives, versus, integrations, feature-specific needs | Match the decision or action closely and provide a direct next step. |

- Classify by the SERP-backed dominant intent, not by a modifier alone.
- Prioritize BOFU opportunities early for SaaS and online-tool sites, while retaining essential MOFU and TOFU pages that build the journey and topic structure.
- Use Ahrefs Search Intent as evidence; keep `Funnel` as the planner's analysis.

## Choose one content type

Use only the following `Content Type` values in `Content Plan`:

| Content Type | Use for |
|---|---|
| `Tool` | A functional generator, checker, calculator, converter, analyzer, or other interactive utility. |
| `Landing Page - Core` | The main product or primary category page. |
| `Landing Page - Feature` | A distinct product capability with its own intent. |
| `Landing Page - Use Case` | A meaningful audience, workflow, or job-specific application. |
| `Landing Page - Variant` | A close product variant that deserves a separate page based on SERP evidence. |
| `Landing Page - Integration` | A product-to-platform integration or connected workflow. |
| `Landing Page - Comparison` | A commercial comparison involving multiple solutions or categories. |
| `Landing Page - Alternative` | An alternative-to-competitor or alternative-to-category page. |
| `Template / Samples` | Reusable templates, examples, prompts, presets, or sample assets that satisfy the query. |
| `Blog - How to` | A step-by-step task or procedure. |
| `Blog - Info` | An explanatory article that does not fit a more specific blog type. |
| `Blog - Listicle` | A list of ideas, methods, examples, resources, or non-review items. |
| `Blog - Best` | A ranked or curated commercial roundup of products or solutions. |
| `Blog - Review` | An in-depth review of one product or service. |
| `Blog - Vs` | A direct comparison between two named products, approaches, or choices. |
| `Blog - Examples` | A page whose main value is a curated set of examples. |
| `Blog - Statistics` | A statistics, data, or trends resource. |
| `Blog - Glossary` | A definition or terminology reference. |
| `Research / Report` | Original research, benchmark, survey, or data-led report. |
| `Case Study` | Evidence of a real customer, project, or outcome. |

Select the type that best matches both the intent and the page formats already succeeding in the Ahrefs SERP. Do not force an informational query into a product landing page or turn a transactional tool query into a blog post.

Include only publishable, indexable website pages in `Content Plan`. Put Newsletter, Podcast, Webinar, YouTube, and other distribution or repurposing ideas in later strategy notes; do not create a primary-plan row for them unless the row represents an indexable website page with independent search demand.

## Keep language and labels consistent

- Write keywords, Topic labels, and planned URL slugs in the required target language.
- Keep `TOFU`, `MOFU`, `BOFU`, `P1`, `P2`, `P3`, Action values, and Content Type values in the standardized English forms above.
- Preserve Ahrefs Search Intent in Ahrefs's standard English labels, including multiple labels when returned.
- Write narrative strategy notes in the user's conversation language.

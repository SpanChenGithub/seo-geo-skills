# Title and Meta Description Pattern Examples

Use these examples only to recognize effective structures. Apply `SKILL.md`, `seo-rules.md`, current page truth, and live SERP evidence before using any pattern.

Never copy an example verbatim. Revalidate every number, year, superlative, comparison, free offer, discount, trial, speed claim, and resource claim. Rewrite each candidate for the target page, language, intent, and editorial length range. Terminal brand suffixes have been omitted to match the skill's current Title rule.

## Title patterns

| Example | Pattern demonstrated |
| --- | --- |
| `30+ Free Meeting Agenda Templates for Teams, Managers, and More` | Meaningful number, verified free resource, and audience scope |
| `How to Edit Videos on Your iPhone, iPad, iPod Touch, or Mac` | Direct how-to intent and supported platform scope |
| `16 Best Screen Recorders for Windows PCs - Free & Paid` | Listicle, platform qualifier, and verified pricing categories |
| `How to Choose the Best (and Worst) Protein Powder in 2026` | Question-led guidance, contrast, and verified freshness |
| `[4 Easiest Ways] How to Record Phone Calls on iPhone in 2026` | Brackets, number, ease angle, and verified year |
| `7 Time Management Tools to Increase Your Productivity` | Number, category keyword, and concrete benefit |
| `How to Play MP4 Files: 10 Steps (with Pictures)` | Task keyword, step count, and included visual resource |
| `The Most Common Zoom Problems and How to Fix Them Now` | Pain point, solution, and urgency when genuinely supported |
| `ChatGPT vs. Microsoft Copilot vs. Gemini: Which Is Best?` | Comparison entities and a decision question |

Extract the structure, not the claims. For example, use a year only when the page is maintained for that year, use `Free` only when access conditions support it, and use `Best` only when the page and intent justify a comparative claim.

## Meta Description patterns

| Example | Pattern demonstrated |
| --- | --- |
| `Need to take notes and record meeting minutes accurately? This step-by-step guide shows you how to take meeting minutes that are clear, concise, and effective.` | Conversational question, natural keyword use, format, and reader outcome |
| `Do you need to change your MacBook name so others recognize your computer? Follow the steps to update it quickly and confirm the new name.` | User problem first, followed by the promised resolution |
| `Convert audio to text with automatic transcription. Upload an audio or video file, review the transcript, and export it in a supported format.` | Tool capability, supported inputs, workflow, and output |
| `Better staff meetings create a better workplace. Learn the dos and don'ts, then use three agenda templates to plan clearer, more productive meetings.` | Benefit-led opening, practical guidance, and included resource |
| `Looking for useful Zoom extensions? Compare 10 plugins and widgets for recording, collaboration, productivity, and easier meeting follow-up.` | Conversational opening, meaningful number, comparison scope, and use cases |

Keep the most valuable page-specific information near the beginning. Summarize or paraphrase the page instead of copying a large passage from its visible content.

## Verified value fragments

Use fragments like these only inside a complete, natural description and only when the target page explicitly verifies the claim:

- `Sign up today and get 50% off.`
- `Try it for 7 days.`
- `Updated for 2026.`
- `Includes 15 insider tips.`

Do not use a fragment merely because it is persuasive. Confirm the exact discount, trial conditions, maintenance date, quantity, eligibility, and availability first.
